# Typography

Extends [Material UI Typography](https://material-ui.com/components/typography/)

`Typography` follows the material API using the system fonts, further extensions allows font styles to be applied.
By default Typography is a `div`, the alternative classes can be provided using the `component` prop. Also see
Shorthands below.

# Headings

Use `Heading` for rendering heading html element e.g. `h1, h2, ...`. Headings should not be used as a substitute for font size changes,
they should be used in line with [best practice](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/Heading_Elements). They have, by default,
responsive size settings, however the sizes can be set independently using the `fontSize` prop and selecting a responsive list of sizes from theme's
fontSizes array, `fontSize={[3,4,5]}`.

## Alternatives

Two alternative typography components are provided to support the use of different fonts, for headings `Display` and normal text `Text`.
The font stack for these can be customized in the `ThemeProvider`. These are intended for use on pages where a simgle font is not sufficient
or engaging enough. Say a help page.

### Display

Traditional heading elements are designed to work best in the main page content.
When you need a heading to stand out, consider using a display heading: a larger, slightly more opinionated heading style.
